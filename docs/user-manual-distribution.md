# ユーザーマニュアル（dist ZIP 配布版）

## 1. このマニュアルの対象

このドキュメントは、配布された `dist` フォルダの ZIP ファイルを使って、Chrome に拡張機能を導入するユーザー向けの手順書です。

## 2. 事前準備

- Google Chrome がインストールされていること
- 配布された ZIP ファイル（例: `x-article-to-markdown-dist.zip`）を受け取っていること

## 3. インストール手順

1. 受け取った ZIP ファイルを任意の場所に展開する  
   - 例: `Downloads/x-article-to-markdown-dist/`
2. Chrome で `chrome://extensions` を開く
3. 右上の「デベロッパーモード」を ON にする
4. 「パッケージ化されていない拡張機能を読み込む」をクリックする
5. 展開したフォルダ内の `dist` ディレクトリを選択する
6. 拡張機能が一覧に表示されればインストール完了

## 4. 基本的な使い方

1. X（旧Twitter）の投稿ページを開く  
   - URL 例: `https://x.com/<user>/status/<tweet_id>`
2. ブラウザ右上の拡張機能アイコンから本拡張を開く
3. 必要に応じてオプションを設定する
   - `Front Matter付与`: Markdown 先頭にメタデータ（YAML形式）を追加します。
   - `画像をローカル保存`: 画像をローカル保存し、Markdown の画像リンクをローカルファイル参照にします。
4. `Markdown生成` を実行する
5. プレビュー内容を確認し、必要に応じて以下を実行する
   - クリップボードにコピー
   - `.md` ファイルとして保存

### オプション利用時の補足

- `Front Matter付与` を有効にすると、出力 Markdown の先頭に `---` で囲まれたメタデータブロックが追加されます。
- `画像をローカル保存` を有効にすると、画像は `images/` フォルダ配下に保存されます。
- 配布先で Markdown と画像をセットで利用する場合は、`.md` ファイルと `images/` フォルダを同じ構成で保管してください。

## 5. よくあるトラブル

### 5.1 「対象ページではありません」と表示される

- X の投稿 URL（`/status/` を含む）を開いているか確認してください。
- タイムライン画面やプロフィール画面では動作しません。

### 5.2 抽出に失敗する

- ページを再読み込みして再実行してください。
- X 側の表示仕様変更や一時的な読み込み不良で失敗する場合があります。
- `画像をローカル保存` が ON の場合、ネットワーク状態によっては一部画像のみ保存に失敗することがあります。

### 5.3 拡張が表示されない

- `chrome://extensions` で本拡張が有効になっているか確認してください。
- 一度削除してから、`dist` を再読み込みしてください。

## 6. 再配布時の注意（配布者向け）

配布者が ZIP を作成する際は、以下を推奨します。

1. 最新コードでビルドする  
   - `npm run build`
2. `dist` フォルダのみを ZIP 化する
3. ZIP 名にバージョンを含める  
   - 例: `x-article-to-markdown-dist-v0.1.0.zip`
4. 本マニュアルを同梱するか、参照先を案内する
