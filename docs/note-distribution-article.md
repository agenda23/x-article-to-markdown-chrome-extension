# 【作業時間を削減】Xの有料級ポストを1クリックでMarkdown化するChrome拡張を作りました

X（旧Twitter）では、日々「これ無料で読んでいいの？」と思うような有料級の情報が流れてきます。  
私自身も、AIエージェント開発や検証で参考になるポストを大量に保存してきました。

ただ、いざ実務で使おうとすると悩みが出ます。

- 後で読み返すときに、Xの画面を行ったり来たりして時間が溶ける
- AIエージェントに渡す前処理（コピー、整形、メタ情報付与）が地味に面倒
- 良記事を集めるほど、管理コストが一気に増える

この問題を解消するために、**Xの投稿ページをMarkdownとして出力するChrome拡張**を作成しました。  
「保存しやすい」「AIに渡しやすい」「再利用しやすい」を優先して設計しています。

## この拡張でできること

結論から言うと、対象ポストのURLを開いて拡張を実行するだけで、AI活用しやすい形に変換できます。

- X投稿ページ（`/status/` URL）をMarkdownへ変換
- 投稿本文、投稿者情報、投稿日時、投稿URLを抽出
- オプションでYAML Front Matterを付与
- オプションで画像をローカル保存し、`images/` 相対パスで参照
- クリップボードコピー / `.md`保存に対応

## なぜ作ったのか（背景）

私は普段、AIエージェントにコンテクストを渡して「要約」「比較」「次アクションの提案」をさせる運用をしています。  
このとき、情報源がブラウザ上のままだと取り回しが悪く、結局は手作業の整形に時間を使ってしまいます。

つまり本質は、

**情報収集がボトルネックではなく、再利用可能な形式に変換する工程がボトルネック**

ということでした。

Markdown化できれば、以下が一気に楽になります。

- ナレッジベースへの保存
- プロンプトへの貼り付け
- Git管理や差分確認
- タグ付けや検索

## 1. インストール手順（配布版）

配布された `dist` ZIP を使って導入します。

1. 受け取ったZIPを任意の場所に展開  
   例: `Downloads/x-article-to-markdown-dist/`
2. Chromeで `chrome://extensions` を開く
3. 右上の「デベロッパーモード」をONにする
4. 「パッケージ化されていない拡張機能を読み込む」をクリック
5. 展開先の `dist` ディレクトリを選択
6. 拡張機能一覧に表示されたら導入完了

---

## 2. 基本的な使い方

1. Xの投稿ページを開く  
   例: `https://x.com/<user>/status/<tweet_id>`
2. ブラウザ右上の拡張機能アイコンから本拡張を開く
3. 必要に応じてオプションを設定する
4. `Markdown生成` を実行する
5. プレビュー確認後、次のいずれかを実行  
   - クリップボードにコピー  
   - `.md` ファイルとして保存

### オプションの使い分け

#### `Front Matter付与`

Markdown先頭にYAML形式のメタ情報を付与します。  
複数記事を管理したい人は、基本ON運用がおすすめです。

#### `画像をローカル保存`

画像を `images/` フォルダへ保存し、Markdown側のリンクをローカル参照に置き換えます。  
オフライン閲覧やアーカイブ用途で強いです。

## 3. 有用な使い方（実運用ノウハウ）

### 使い方1: AIエージェントにそのまま渡す

Markdown化した内容を、AIエージェントの入力コンテキストへ貼り付けます。  
URLだけ渡すよりも、本文とメタ情報が整理されているため、回答品質が安定しやすくなります。

### 使い方2: テーマ別フォルダでナレッジ化する

例:

- `knowledge/x/ai-agent/`
- `knowledge/x/marketing/`
- `knowledge/x/product/`

このようにテーマ別に分けると、後からの再利用速度が大きく上がります。

### 使い方3: Obsidianで一元管理する（最重要）

この拡張との相性が最も良い運用の1つが、**Obsidian管理**です。  
理由はシンプルで、X投稿の断片的な知見を「リンクできる知識」に変えられるからです。

#### なぜObsidian管理が強いのか

- Markdownがそのまま第一級データとして扱える
- ローカル保存なので長期アーカイブに向く
- フォルダ、タグ、リンク、検索を組み合わせて高速に再利用できる
- AIに渡す前の前処理がほぼ不要になる

#### 推奨Vault構成（そのまま使える例）

```text
vault/
  inbox/
    x/
      2026-04/
  knowledge/
    ai-agent/
    marketing/
    product/
  daily/
```

最初は `inbox/x/` に全部入れてOKです。  
週次レビュー時に、価値が高いノートだけ `knowledge/` へ移動すると運用が崩れません。

#### Front Matterを活かした管理方法

本拡張の `Front Matter付与` をONにしておくと、後工程がかなり楽になります。  
以下のような項目でそろえると、検索と抽出が安定します。

- `source: x`
- `author: <投稿者>`
- `url: <投稿URL>`
- `created: <投稿日時>`
- `tags: [x, ai-agent, 保存用]`

この形式で蓄積すると、Obsidian内検索や各種プラグインで絞り込みしやすくなり、  
「あのとき見た投稿どこだっけ？」がほぼ消えます。

#### 実運用フロー（5分で回せる）

1. 気になったX投稿を本拡張でMarkdown化
2. `inbox/x/` に保存（画像ONなら `images/` も同階層で保持）
3. Daily Noteに1行メモ（なぜ保存したか）
4. 週1回だけ見直し、重要ノートを `knowledge/` へ移動
5. 関連ノート同士を内部リンクで接続

この手順にすると、単発メモが「使える知識グラフ」に育っていきます。

#### AIエージェント連携まで含めた使い方

Obsidianで整理したノート群は、そのままAIエージェントの入力素材になります。  
例えば、同一テーマのノートを3〜10件まとめて渡し、

- 共通主張の抽出
- 発信者ごとの差分整理
- 次の検証タスクの提案

を依頼すると、インプットが具体的な行動に変わります。  
「保存して終わり」ではなく、「判断精度を上げる資産」になるのがこの運用の価値です。

### 使い方4: 週次レビューを自動化する

1週間分のMarkdownをまとめてAIに渡し、

- 今週の重要トピック
- 実行すべきアクション
- 追うべき発信者

を抽出させると、インプットが意思決定に直結します。

### 使い方5: 自分専用の「学習データセット」を作る

刺さった投稿だけを蓄積し、ジャンル別に育てていくと、  
将来的に「自分の判断軸に最適化された参照データ」として機能します。

## 4. よくあるトラブルと対処

### 「対象ページではありません」と表示される

- `/status/` を含む投稿URLを開いているか確認
- タイムラインやプロフィール画面では動作しません

### 抽出に失敗する

- ページを再読み込みして再実行
- X側の表示仕様変更や一時的な読み込み不良の可能性あり
- 画像保存ON時は、通信状態により一部画像保存が失敗する場合あり

### 拡張が表示されない

- `chrome://extensions` で有効化されているか確認
- 一度削除して `dist` を再読み込み
